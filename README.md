# metadata-retrieval-playground

## v3

Known from ghsync.

Note: there are resources that we were not downloading in the `deep` mode. PRs have:
- normal issue comments
- reviews (groups of review comments)
- review comments (each comment made on the diff)

The code for `ghsync deep` only downloaded normal issues. The code in this repo fetches all 3.

Time examples:
- https://github.com/carlosms-test-org/test-repo `go run cmd/metadata/main.go v3  --owner=carlosms-test-org --name=test-repo`:
  - 7.1s
- https://github.com/src-d/gitbase `go run cmd/metadata/main.go v3  --owner=src-d --name=gitbase`:
  - 22m44.2s

Code is mostly copied from `ghsync` deep. Places where we can speed up:
- query issue comments for the repo, instead of for each issue
- use the list endpoints for issues and PRs if we think the lost data is not relevant

### list vs individual endpoint contents

In `ghsync shallow` we used the list endpoints to get the data for issues by pages, instead of calling the individual endpoint for each issue. This section uses the `--log-http` option to create json files and compare them.

## migration api

Repositories can be downloaded in a tar.gz to be migrated from github.org to a github enterprise instance.

https://developer.github.com/v3/migrations/orgs/

But the tar.gz file contains also the git repository, not just the metadata.

One big problem is that the token must be from an organization owner account.

There is some waiting time before the file can be downloaded. For a small repository this means the download time is quite higher than using `v3`.

Time for `go run cmd/metadata/main.go migration  --owner=carlosms-test-org --name=test-repo`: 16.8s

### contents comparison

Sample comparison for https://github.com/carlosms-test-org/test-repo.

Concussion: the json downloaded from the migration tar.gz are a **similar** schema, but with much less fields used compared to the json returned by the API.

The JSON from the migration API cannot be automatically unmarshaled into `google/go-github` objects. See `cmd/migration-unmarshal/main.go`:

```
$ go run cmd/migration-unmarshal/main.go
```

- json for issues copied to [./samples/issues-migration.json](./samples/issues-migration.json), and edited to have the same order as the other json files.
- json for prs copied to [./samples/prs-migration.json](./samples/prs-migration.json),
